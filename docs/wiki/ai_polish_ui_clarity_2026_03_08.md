# AI 润色设置页 UI 优化 (2026-03-08)

## 背景

AI 润色 Tab 页存在两个 UX 问题：

1. **"AI 润色"开关歧义**：第一个分组标题与 Tab 名称均为"AI 润色"，开关标签仅写"启用"，用户误以为这是整页总开关。实际上它只控制 LLM 改写，下方的专业词汇功能是独立的。
2. **缺少组合说明**：LLM 改写和专业词汇是两个正交开关，有 4 种组合行为，用户无法直观理解。

## 改动

### 分组重命名

- 第一个 SettingsGroup 标题：`AI 润色` → `LLM 智能改写`
- Tab 名称保持 "AI 润色" 不变（作为整页概念伞）

### 2x2 组合矩阵说明

在 LLM 智能改写区域和专业词汇区域之间插入灰色信息框：

| LLM 改写 | 词典 | 行为 |
|-----------|------|------|
| ✓ | ✓ | 术语注入 LLM，智能纠错 |
| ✓ | ✗ | 纯 LLM 润色 |
| ✗ | ✓ | 词典精确替换（离线可用） |
| ✗ | ✗ | 原始 ASR 输出 |

## 涉及文件

- `lib/ui/settings_page.dart` — 分组标题 + 矩阵说明 UI
- `lib/l10n/app_zh.arb` / `app_en.arb` — 新增 `llmRewrite`、`aiPolishMatrix` 国际化键
- `lib/l10n/generated/` — 自动生成的 l10n 文件
